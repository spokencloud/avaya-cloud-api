Public API to create a subscription

Parameters
endPointThe HTTP/HTTPS endpoint where data will be sent
dataDeliveryFormatFormat for encoding the data. Valid values are "CSV" and "JSON". If unspecified, the format is CSV
dataSourceTypeThe name of the data source for the subscription. Valid values are: ECH, HAGLOG, HAGENT, RT_AGENT_STATE, RT_DID_STATE, RT_SKILL_STATE, RT_VDN_STATE
frequencyInMinutesThe frequency of how often data is sent. If the value is zero (default) data is sent as quickly as possible. Other values (ex: 10-mintues) try to batch data for up to Frequency units.
startTimeThe start time of the subscription. When created, the data will be sent starting from StartTime.
basicAuthUsernameIf set, the producer will use the indicated BasicAuth username and password to connect to the endpoint. BasicAuth will only be used if the protocol is https.
basicAuthPassword
maxPostSizeThe maximum size (in bytes) of a POST request that the endpoint can accept. The producer will always flush before exceeding this number of bytes. The default (0) is unlimited.
subAccountAppIdSubAccountAppId for which the subscription data is needed. If login user is sysadmin, then it can fetch data for all subaccount also with subaccountappid as 'ALL'
Returns
Created Subscription object
